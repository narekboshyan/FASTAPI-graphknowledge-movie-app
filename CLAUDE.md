## Commands

Full stack (API + Neo4j) via Docker Compose — requires `NEO4J_USER` and `NEO4J_PASSWORD` in `.env`:

```
docker compose up -d           # start api + neo4j
docker compose logs -f neo4j   # tail Neo4j logs
docker compose logs -f api     # tail API logs
docker compose down            # stop + remove containers (volumes preserved)
docker compose down -v         # also wipes graph data volume
```

Local dev without Docker (Neo4j must be reachable at `NEO4J_URI`):

```
pip install -r requirements.txt
fastapi dev main.py            # hot-reload on :8000
```

No test suite, linter, or formatter is configured in this repo.

Neo4j browser: http://localhost:7474 · Bolt: `bolt://localhost:7687` · API docs: http://localhost:8000/docs

## Architecture

FastAPI service exposing a Neo4j-backed movie knowledge graph. Single-process app, no ORM — routers run raw Cypher against a shared `neo4j.GraphDatabase` driver.

- `main.py` — registers routers and installs an HTTP middleware that times every request and logs via Rich with `[success]`/`[warning]`/`[error]` theme tags. Closes the driver on shutdown.
- `db.py` — module-level singleton `driver`. Imported everywhere; do not construct additional drivers. Env: `NEO4J_URI`, `NEO4J_USER`, `NEO4J_PASSWORD` (loaded via `python-dotenv`).
- `logger.py` — Rich-themed logger named `movie-api`. Also redirects `uvicorn`, `uvicorn.access`, `uvicorn.error` through the same handler so all output is one stream. Log strings use theme markup (e.g. `[success]✓[/]`) — preserve this style in new log lines.
- `models.py` — Pydantic request/response models (`Movie`, `Person`). Keep graph node properties and Pydantic fields aligned.
- `routers/` — one module per resource, all auto-included in `main.py`:
  - `movies.py`, `people.py` — CRUD on `:Movie` / `:Person` nodes.
  - `relationships.py` — `ACTED_IN`, `DIRECTED` edges. Uses `MATCH ... MERGE` so both endpoints must exist; returns 404 if either node is missing.
  - `seed.py` — bulk graph population.

Graph model: `(:Person)-[:ACTED_IN|DIRECTED]->(:Movie)`. Node identity is by `name` (Person) and `title` (Movie) — there are no unique constraints in code, so callers must not create duplicates.

Cypher pattern used throughout: open a `with driver.session() as session:` block, run a parameterized query, call `.single()`, and raise `HTTPException(404)` when no record comes back. Follow this pattern for new endpoints rather than introducing a repository/service layer.

## Container notes

`compose.yaml` bind-mounts the repo into `/app` for live editing, but the Dockerfile `CMD` runs `fastapi run` (no reload). For iterative work either use `fastapi dev` locally against the Compose Neo4j, or override the API service command.
